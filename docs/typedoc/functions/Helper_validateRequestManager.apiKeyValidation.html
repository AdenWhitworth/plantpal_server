bail

  • bail(opts?): ValidationChain

  • Stops running validations if any of the previous ones have failed.

    Useful to prevent a custom validator that touches a database or external API from running when you know it will fail.

    May be used multiple times in the same validation chain if desired.

    Parameters

    • Optionalopts: BailOptions

    Returns ValidationChain

    the current validation chain

    Example

    check('username')
       .isEmail()
       // If not an email, stop here
       .bail()
       .custom(checkDenylistDomain)
       // If domain is not allowed, don't go check if it already exists
       .bail()
       .custom(checkEmailExists)

  • if(condition): ValidationChain

  • Adds a condition on whether the validation should continue on a field or not.

    Parameters

    • condition: ContextRunner | CustomValidator

      may be either

      • a custom validator-like function, which must truthy or a promise that resolves to continue validation. If the return value is falsy, a promise that rejects, or if it throws, validation will stop.
